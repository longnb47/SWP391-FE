## Brand & Style

The design system is engineered for a high-velocity, intelligence-driven file management experience. It targets power users and creative professionals who require an energetic yet structured environment to manage vast amounts of data. The brand personality is **Intelligent, Efficient, and Energetic**, balancing the warmth of a vibrant primary palette with the precision of a minimalist, modern interface.

The visual style leans into **Minimalism with a Tactile Edge**. By utilizing heavy whitespace and a refined color palette, the system ensures that user content and AI-driven insights remain the focal point. Surfaces are treated with subtle depth to indicate interactivity, creating an atmosphere that feels both professional and cutting-edge.

## Colors

The palette is anchored by a **Vibrant Orange (#FF6B00)**, used strategically for primary actions, AI status indicators, and key branding moments. This high-energy hue is balanced by a sophisticated range of **Warm Grays** and crisp whites to prevent visual fatigue during long work sessions.

- **Primary**: Used for the "North Star" actions: Upload, AI Chat triggers, and active navigation states.
- **Secondary**: A deep, near-black gray used for high-contrast typography and icon outlines.
- **Tertiary**: A softer orange tint used for secondary accents, hover states on light backgrounds, and data visualization.
- **Neutral/Surface**: Warm grays (derived from the primary hue's complement) provide the foundation for sidebars, file containers, and background layers, ensuring the interface feels approachable rather than clinical.

## Typography

This design system utilizes **Inter** for its exceptional legibility and systematic feel. The type hierarchy is designed to guide the eye through dense file structures and complex AI interactions.

- **Headlines**: Use tighter letter-spacing and semi-bold weights to create a strong visual anchor for page titles and folder names.
- **Body Text**: Optimized for readability with a generous line height (1.5x) to ensure metadata and file descriptions are easily scannable.
- **AI Feedback**: Text generated by the AI interface should utilize `body-md` but may incorporate a monospaced font for technical file paths or metadata strings to differentiate machine-generated content from user input.

## Layout & Spacing

The layout employs a **Hybrid Fixed-Fluid Grid**. The sidebar and AI chat panels are fixed-width to maintain consistent utility, while the central file explorer is fluid, allowing users to maximize their workspace.

- **Sidebar**: Positioned on the left, acting as the primary navigation hub.
- **Main Content**: Uses a 12-column grid for dashboard views and a flexible flexbox layout for file list/grid views.
- **AI Interface**: An expandable drawer on the right side of the screen, or a persistent bottom-anchored bar for quick queries.
- **Spacing Rhythm**: Based on an 8px base unit. Gaps between file cards should be `16px` (2 units), while internal padding for cards and modals should be `24px` (3 units) to maintain a spacious, professional feel.

## Elevation & Depth

To achieve a modern, minimalist look without sacrificing usability, the system uses **Ambient Shadows** and **Tonal Layering**.

- **Level 0 (Background)**: Pure white or the lightest warm gray (#F9F9F8).
- **Level 1 (Cards/Sidebar)**: Raised slightly with a very soft, diffused shadow (0px 4px 20px rgba(0,0,0,0.04)).
- **Level 2 (Modals/Popovers)**: Higher elevation with a more pronounced shadow and a subtle 1px border in a light warm gray to define edges against white backgrounds.
- **AI Elements**: The AI chat interface utilizes a subtle **Backdrop Blur** (12px) when overlaid on content, signaling its "intelligent" and secondary layer status.

## Shapes

The shape language is defined by **Soft Geometricity**. A consistent corner radius creates a friendly and modern aesthetic that avoids the sharpness of traditional enterprise software.

- **Small Elements (Checkboxes, Tags)**: 4px radius.
- **Standard Elements (Buttons, Inputs, File Cards)**: 8px to 12px radius.
- **Large Elements (Modals, AI Panel)**: 16px to 24px radius.
- **Pill Shapes**: Used exclusively for status indicators (e.g., "Uploading") and secondary action chips to differentiate them from primary rectangular buttons.

## Components

### Sidebar Navigation
The sidebar uses a clean, vertical list with 12px rounded hover states. Icons are 20px, stroke-based, using the `secondary_color`. Active states use a 4px vertical "pill" indicator in `primary_color` on the left edge.

### File Cards & List Items
Cards feature a large icon or thumbnail, a primary title, and secondary metadata in `label-md`. In list view, items are separated by a subtle 1px divider. Hovering over a file reveals a hidden "AI Quick-Action" button.

### AI Chat Interface
The interface uses a dedicated input field with a subtle orange glow when focused. Message bubbles are distinct: user messages are light gray, while AI responses are white with a thin primary-colored border.

### Primary Buttons
High-contrast `primary_color` background with white text. Buttons feature a slight 2px bottom-shadow to give them a "clickable" tactile feel.

### Input Fields & Search
Search bars are large (48px height) with a 12px radius. They utilize a "Warm Gray" background that transitions to white on focus, accompanied by a subtle orange outer glow.
